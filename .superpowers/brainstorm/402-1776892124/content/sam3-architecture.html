Architektur mit SAM 3 — vereinfacht

SAM 3 hat einen eingebauten DETR-Detektor, der auf Text konditioniert ist. Zwei Modelle werden eins.

Option B (alt) — Zwei-Stufen-Pipeline
Text: "solar panel"
↓ 1) GroundingDINO
→ Bounding Boxes + Scores
↓ 2) SAM 2 (Box-Prompt)
→ Pixel-Masken
↓ NMS + Polygonisierung
• ~2 Modelle im VRAM
• 2 Inferenzstufen pro Kachel
• Zwei Fehlermodi
Option B' (neu) — SAM 3 end-to-end
Text: "solar panel"
↓ SAM 3 (DETR + Segmenter)
→ Masken + Boxes + Scores
↓ NMS + Polygonisierung
✓ 1 Modell (848 M Params, ~4–6 GB)
✓ 1 Inferenzstufe pro Kachel
✓ Ein Fehlermodus statt zwei
✓ Detektor & Segmenter co-trained

Gesamt-Pipeline

Datenfluss: Text + BBox → GeoPackage
① UI (FastAPI): Nutzer gibt Prompt + BBox ein → POST /jobs
② WCS-Client (Bayern LDBV): lädt DOP20-GeoTIFF, speichert Affintransformation
③ Geo-Tiler (rasterio): 1024×1024 Kacheln mit 256 px Overlap, jede Kachel kennt ihre Pixel-Offsets
④ SAM 3 Inferenz: pro Kachel → Masken + Boxes + Scores
⑤ Merger: Pixel-Masken → lokale Polygone (Shapely) → in DOP-Pixelraum projizieren → Polygon-NMS via IoU
⑥ Georeferenzierung: Affintransformation × Polygon-Pixel → EPSG:25832-Koordinaten (GeoPandas)
⑦ UI-Preview: GeoJSON-Endpoint → Leaflet-Karte → Nutzer validiert (✓/✗ pro Polygon)
⑧ Export: gefilterte Polygone → GeoPackage (.gpkg)
Ein Caveat zu SAM 3

SAM 3 erfordert Python 3.12+, PyTorch 2.7+, CUDA 12.6+. Das ist ein moderner Stack — deine RTX 4070 unterstützt CUDA 12.6 problemlos, aber die Treiber müssen aktuell sein. Das Modell ist neu (Nov 2025), Community-Tutorials sind rar, wir verlassen uns auf die offizielle Doku. Falls die Lizenz (SAM License) problematisch werden sollte, ist ein Fallback auf Grounded-SAM-2 mit ~1 Tag Arbeit machbar.
